General:

- snake case all table and column names in order to avoid issues
- set ALL fields as varchar at first
- ensure column names are not reserved keywords (best way to do this is to manually type column names into pgadmin to let the application highlight words that may be reserved)
- drop tables and recreate tables with renamed column names to avoid SQL keyword issues
- run COUNT query on database to ensure each table has the right number of columns
- run COUNT query on each table to ensure the right amount of rows are present
- run queries on each column to determine data type
- for integers and floating point numbers, run queries to determine what data type to use and to determine how many digits are required before and after the decimal point
- run queries to determine if any columns are null, then make judgment call on what type to convert to by looking at the column name
- determine which columns have completely unique values, and using this info combined with the column name, determine if the column is supposed to be unique
- check if a column has null values, and if not, combine with the column name to determine if column should be non-nullable
- determine which columns are primary keys and foreign keys based on previous info
- create primary keys for tables that do not have a unique column relevant to the table names
